# Creating and Managing Lessons

## Overview

Lessons are the core content of your course. This guide covers how to create different types of lessons and manage them effectively.

## Accessing Lesson Creation

1. Navigate to your course editing page
2. Go to the "Curriculum" section
3. Select a chapter where you want to add a lesson
4. Click "Add Lesson" or the "+" button

## Lesson Types

### Video Lessons

Video lessons are ideal for demonstrations, explanations, and visual learning.

**Creating a Video Lesson:**

1. Select "Video" as the lesson type
2. Upload a video file or provide a video URL
3. Add a lesson title
4. Optionally add lesson content/description
5. Set the lesson position within the chapter
6. Save the lesson

**Video Requirements:**

- Supported formats: MP4, MOV, AVI, and other common video formats
- Recommended resolution: 1280x720 (720p) or higher
- Keep videos between 2-7 minutes for optimal engagement
- Ensure good audio quality

**Video Best Practices:**

- Start with a brief introduction
- Use clear, concise explanations
- Include visual aids and demonstrations
- End with a summary or key takeaway

### Article Lessons

Article lessons are perfect for text-based content, reading materials, and detailed explanations.

**Creating an Article Lesson:**

1. Select "Article" as the lesson type
2. Enter a lesson title
3. Write your content using the markdown editor
4. Format text with headings, lists, code blocks, etc.
5. Set the lesson position
6. Save the lesson

**Article Best Practices:**

- Use clear headings and subheadings
- Break content into digestible paragraphs
- Include examples and code snippets where relevant
- Use formatting (bold, italic, lists) to improve readability
- Keep articles focused on a single topic

### Quiz Lessons

Quiz lessons help assess student understanding and reinforce learning.

**Creating a Quiz Lesson:**

1. Select "Quiz" as the lesson type
2. Enter a quiz title
3. Add questions with multiple choice options
4. Mark the correct answer(s) for each question
5. Set the lesson position
6. Save the lesson

**Quiz Best Practices:**

- Create clear, unambiguous questions
- Provide 3-5 answer options per question
- Include explanations for correct answers
- Mix easy, medium, and challenging questions
- Test understanding, not just memorization

## Editing Lessons

### Modifying Lesson Content

1. Navigate to the Curriculum section
2. Click on the lesson you want to edit
3. Update the title, content, or type
4. Save your changes

### Reordering Lessons

- Adjust the position field to change lesson order
- Or use drag-and-drop if available
- Ensure logical flow within each chapter

### Deleting Lessons

1. Click the delete/trash icon on the lesson
2. Confirm deletion in the dialog
3. Note: This action cannot be undone

## Lesson Attachments

### Adding File Attachments

1. While creating or editing a lesson
2. Use the file upload component
3. Upload supporting materials (PDFs, documents, code files, etc.)
4. Students can download these attachments when viewing the lesson

**Supported File Types:**

- Documents: PDF, DOC, DOCX
- Code files: Various programming language files
- Images: JPG, PNG, GIF
- Other: ZIP archives, etc.

## Lesson Completion Requirements

For students to complete a lesson, they typically need to:

- **Video Lessons**: Watch the video (may require watching a certain percentage)
- **Article Lessons**: Read through the content
- **Quiz Lessons**: Complete the quiz and achieve a passing score

## Tips for Effective Lessons

- **Clear Objectives**: Start each lesson with what students will learn
- **Engaging Content**: Use varied formats to maintain interest
- **Practical Examples**: Include real-world applications
- **Progressive Difficulty**: Build complexity gradually
- **Regular Assessments**: Mix content with quizzes to reinforce learning
- **Supporting Materials**: Provide downloadable resources when helpful

## Preview Lessons

1. Use the preview feature to see how lessons appear to students
2. Test video playback, article formatting, and quiz functionality
3. Make adjustments based on the preview
